How this controller works — the teaching points

Feed-forward does most of the work

Before any error is measured, FF = offset + slope·target pushes out the PWM the motor needs at that speed. Match the sliders to the motor (offset 50, slope 0.15) and feed-forward alone nails the target — open-loop.

The integral erases the leftover

Whatever FF and P can't cover, the I term slowly accumulates until steady-state error is zero. Apply a load and watch I climb to compensate.

Anti-windup freezes I at the limits

Integration only happens while 0 < output < 255. Saturate the output and the I term stops growing — so it doesn't "wind up" and overshoot on recovery.

Derivative on the measurement

D = −Kd·(Δspeed)/dt, not on the error — so a step change in target doesn't cause a derivative "kick". D damps the response to real speed changes only.

Smoothing tames the encoder

At 40 Hz each encoder count is ~34 RPM, so the raw reading is a staircase. The EMA smoothing S calms it — trade noise for lag.

Try this

Set Ki = 0 and mistune FF → steady error. Add Ki → it closes. Apply load → sag & recover. Push Kp high → oscillation; add a little Kd to damp it.
